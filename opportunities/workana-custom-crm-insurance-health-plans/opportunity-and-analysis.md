# Workana Opportunity: CRM Personalizado para Seguros e Planos de Saúde

## Original opportunity

Estamos buscando um desenvolvedor ou equipe para criar um sistema CRM (Customer Relationship Management) personalizado. O sistema deve ser robusto e intuitivo, focado na gestão completa de clientes e processos de vendas.

Funcionalidades essenciais:

- Gestão de dados de clientes, com nome, endereço, contato e outros dados relevantes.
- Campo para anexos, permitindo documentos, contratos ou outros arquivos importantes em cada registro de cliente ou venda.
- Registro de produtos e vendas, com produtos vendidos, valores e comissões associadas.
- Dashboard de desempenho, com vendas mensais, ganhos totais, despesas fixas e novos clientes adquiridos por mês.
- Sistema de pesquisa para buscar e filtrar clientes e vendas.
- Compatibilidade mobile por design responsivo.

O cliente atua no ramo de seguros e planos de saúde e busca uma solução que otimize processos internos e forneça insights claros sobre desempenho de vendas e base de clientes.

## Fit with Fabricio Santana's positioning

Esta oportunidade é um bom encaixe porque combina sistema interno web, CRM, modelagem de dados, armazenamento de arquivos, regras comerciais, dashboard, busca, responsividade, segurança e implantação de uma solução operacional. Ela se alinha ao posicionamento de entrega com escopo controlado, arquitetura, testes, banco de dados, cloud, handoff e propriedade do código pelo cliente.

Pontos de força relevantes:

- 20+ anos em engenharia de software, liderança técnica, arquitetura, backend, banco de dados, testes automatizados, cloud e sistemas críticos.
- Experiência com sistemas internos, regras de negócio, segurança, integrações, dashboards e manutenção.
- Capacidade de transformar uma lista ampla de funcionalidades em um primeiro release operacional controlado.
- Abordagem comercial baseada em discovery, pacotes fixos, critérios de aceite, documentação, demos e handoff.

## Opportunity assessment

O desafio central não é apenas construir cadastros de clientes. O problema real é criar um CRM operacional para seguros e planos de saúde em que dados de clientes, anexos, produtos, vendas, comissões, despesas e indicadores formem um fluxo confiável para gestão comercial.

Os principais riscos são:

- modelo de dados insuficiente para representar clientes, contatos, dependentes, contratos, apólices, planos, propostas, vendas e renovações;
- anexos sem controle de acesso, organização, limite de tamanho, política de backup ou rastreabilidade;
- regras de comissão e receita mal definidas, gerando divergência em relatórios;
- dashboard com números bonitos, mas sem definição clara de origem, período, status e fórmula;
- busca sem filtros realmente úteis para operação comercial;
- falta de permissões por perfil, principalmente se houver vendedores, gestores e administradores;
- subestimar LGPD e segurança em documentos contratuais, dados pessoais e informações ligadas a planos de saúde;
- tentar colocar automações, financeiro completo, WhatsApp, renovações e integrações externas no primeiro ciclo sem delimitar prioridade.

A primeira entrega deve ser uma versão operacional controlada, com cadastro de clientes, vendas, produtos, anexos, dashboard inicial, busca e telas responsivas. A Discovery Sprint deve definir exatamente quais dados e fluxos entram nesse release para evitar um CRM amplo demais para um único ciclo.

## Recommended commercial framing

O caminho recomendado é vender a Discovery Sprint de 1 semana como próximo passo imediato. Ela deve produzir:

- mapa do fluxo comercial atual;
- modelo inicial de dados;
- lista priorizada de telas e funcionalidades;
- definição do primeiro release operacional;
- regras de comissão, ganhos, despesas e indicadores;
- requisitos de segurança, permissões, anexos e LGPD;
- critérios de aceite;
- recomendação do pacote de implementação.

Estimativa sugerida:

- Próximo passo imediato: R$ 3.000 para Discovery Sprint.
- Pacote provável depois da discovery: R$ 24.000 para 4 semanas de `Product Acceleration`, se o primeiro release for controlado.
- Possível revisão para R$ 36.000 em `Scale & Intelligence` se houver exigência forte de auditoria, permissões avançadas, regras complexas, automações, integrações ou maior governança de dados.

## Key questions before final commitment

- Quem usará o CRM: vendedores, gestores, administradores ou outros perfis?
- Quais dados de cliente são obrigatórios no primeiro release?
- O CRM precisa controlar apenas clientes e vendas ou também propostas, apólices, contratos, renovações e dependentes?
- Como são calculadas as comissões por produto, vendedor, venda, parcela ou recorrência?
- Quais despesas fixas entram no dashboard e de onde esses dados serão alimentados?
- Os anexos terão documentos sensíveis, contratos assinados ou informações de saúde?
- Quais filtros de busca são essenciais para a operação diária?
- O sistema precisa importar dados existentes de planilhas ou outro CRM?
- O primeiro release precisa ter integração com WhatsApp, e-mail, ferramentas de pagamento, seguradoras ou operadoras?
- Já existe orçamento aprovado e data-alvo para a primeira versão em uso?
