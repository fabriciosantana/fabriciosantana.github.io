This is not only a website redesign. The key issue is rebuilding a public bilingual company website while also creating a secure employee portal, which means public content, brand implementation, authentication, internal workflows, and a friendly editing experience all need to be designed as one coherent system.

I would start with a one-week Discovery Sprint to define the boundary between the public site and the portal, map the content and current functionality, confirm the bilingual CMS/editing workflow, and produce a practical implementation plan with architecture recommendation, acceptance criteria, security model, and validated first-release scope.

From there, the work can move directly into a fixed implementation cycle for the agreed first operational release, including the public site foundation, portal foundation, and the content-management model your team will use after launch.
